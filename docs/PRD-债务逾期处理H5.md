# 债务逾期处理 H5 产品需求文档 PRD

## 1. 文档信息

- 项目名称：OverdueFree
- 前端应用名：逾期上岸
- 当前版本：V0.1 Demo 需求版
- 文档日期：2026-05-10
- 目标形态：手机 H5
- 当前阶段：需求确认与 Demo 规划

## 2. 项目背景

朋友团队从事逾期债务处理相关业务，希望通过一个手机 H5 产品承接潜在客户，完成手机号获取、债务情况收集、企业微信咨询引导，并让老板在后台查看和导出线索。

第一版不追求完整债务管理系统，而是先做一个可演示、可小范围试用、未来可正式上线扩展的获客型产品。

## 3. 产品目标

### 3.1 核心目标

1. 用户进入 H5 后必须手机号登录，系统获取手机号。
2. 用户可浏览首页、成功案例、资讯、个人中心等页面，形成完整产品感。
3. 用户可通过 AI 债务咨询、规划评估页提交更完整的债务资料。
4. 用户点击咨询、领取方案等按钮时展示统一企业微信二维码，并记录行为。
5. 老板和少数管理员可登录后台查看客户、线索、行为，并导出 Excel。
6. 后台可管理首页素材、视频、企业微信二维码、资讯、成功案例、管理员账号。

### 3.2 第一版成功标准

1. 在 Windows 本地电脑可完整启动并演示。
2. 客户端 H5 可完成登录、浏览、AI 留资、规划评估、查看二维码、使用利率计算器。
3. 后台可查看仅登录客户、已提交资料线索、二维码查看行为。
4. 后台可筛选并自定义字段导出 Excel。
5. 后台可新增管理员、上传素材、维护资讯和成功案例。

## 4. 用户角色

### 4.1 客户

已逾期或快逾期的用户，可能有网贷、信用卡、信用贷、车贷等债务问题。客户希望了解是否可以延期、协商、减免、规划还款等。

### 4.2 老板账号

后台最高权限用户。可以查看和导出线索，管理素材、资讯、案例，新增或禁用管理员，重置管理员密码。

### 4.3 普通管理员

可查看线索、筛选导出、管理素材、资讯、案例。不可管理管理员账号。

## 5. 平台与技术约束

### 5.1 前端

- 技术方向：uni-app + Vue3
- 第一版发布形态：H5
- 未来扩展方向：微信小程序、Android、iOS、鸿蒙

### 5.2 后端

- Java 版本：JDK 11
- 后端框架：Spring Boot 2.7.x
- 架构：单体架构
- 构建工具：Maven
- 数据库：MySQL
- 持久层：MyBatis-Plus
- Redis：第一版不接入

### 5.3 时间与时区

整个项目统一使用北京时间。

1. Spring Boot 应用时区使用 `Asia/Shanghai`。
2. MySQL Docker 使用 `Asia/Shanghai`，并设置默认时区为 `+08:00`。
3. JDBC 连接使用 `serverTimezone=Asia/Shanghai`。
4. 数据库业务时间字段按北京时间写入和读取。
5. 前台、后台、Excel 导出展示的时间均以北京时间为准。

### 5.4 本地演示环境

- 操作系统：Windows
- MySQL：通过 Docker 启动
- 后端端口：8080
- 前端 H5 开发端口：5173
- MySQL 端口：3306
- 上传目录：默认使用可配置的本地目录，例如 `C:/OverdueFree/uploads`

## 6. Git 与开发规范

1. 使用本地 Git 仓库。
2. `master` 为正式稳定分支，只放测试通过、可上线代码。
3. 新功能必须从 `master` 拉功能分支开发。
4. 功能分支完成并测试通过后，合并到 `master` 前必须等待确认。
5. 提交信息建议使用规范前缀，例如：
   - `feat: 新增客户手机号登录`
   - `fix: 修复移动端底部按钮遮挡`
   - `chore: 初始化项目结构`

## 7. 客户端 H5 需求

### 7.1 登录

1. 用户首次进入 H5 必须手机号登录。
2. 手机号按中国大陆 11 位手机号格式校验。
3. 验证码为模拟验证码。
4. 点击获取验证码后，页面直接显示本次模拟验证码，方便测试。
5. 登录成功后保持登录状态，默认有效期 7 天。
6. 登录有效期作为数据库配置项保存。
7. 换设备访问时需要重新验证码登录。
8. 个人中心不展示手机号，但提供退出登录按钮。

### 7.2 首页

首页按对标截图风格实现，并在不偏离风格的前提下做专业化优化。

默认包含：

1. 顶部宣传视频入口。
2. AI 债务咨询师入口。
3. 网贷利率计算器入口。
4. 规划优化债务入口。
5. 服务流程展示。
6. 底部固定主按钮。

视觉基调：

1. 浅暖色背景。
2. 红色主按钮。
3. 大圆角卡片。
4. 底部 Tab 导航。
5. 字体、间距、层级统一优化。

### 7.3 底部导航

底部 Tab 顺序：

1. 成功案例
2. 主页
3. 资讯
4. 个人中心

默认进入主页。

### 7.4 企业微信二维码

1. 全站使用统一企业微信二维码。
2. 首页主按钮、成功案例申请按钮、资讯详情领取按钮、个人中心底部 CTA 等默认展示企业微信二维码。
3. 展示二维码时记录一次用户行为。
4. 第一版不直接跳转企业微信，后续再扩展。

### 7.5 AI 债务咨询

第一版 AI 咨询不接大模型，使用固定聊天式留资流程。

默认流程：

1. 欢迎语。
2. 询问姓氏。
3. 询问地区。
4. 询问债务金额。
5. 选择债务类型。
6. 询问补充描述。
7. 提交成功。
8. 展示企业微信二维码。

提交后生成一条线索记录，来源为 `AI咨询`。

### 7.6 规划优化债务 / 免费方案

规划优化债务入口和免费方案入口共用结构化评估页。

默认字段：

- 必填：姓氏、地区、债务金额、债务类型
- 选填：年龄段、工作情况、征信情况、月收入、月开销、补充描述
- 手机号：从登录信息带入，不重复填写

提交后生成一条线索记录，来源为 `规划评估`。

### 7.7 网贷利率计算器

字段参考对标截图：

1. 分期金额
2. 分期期数
3. 一次性手续费
4. 每期手续费或利率

功能要求：

1. 点击计算后展示估算年化利率。
2. 展示提示：计算结果仅供参考。
3. 不保存计算记录。
4. 记录一次“使用计算器”的行为。

### 7.8 成功案例

前台列表按卡片展示。

字段展示：

1. 称呼
2. 脱敏手机号
3. 逾期平台
4. 逾期金额
5. 处理方案
6. 按钮：我也申请

交互：

1. 点击卡片进入简单详情页。
2. 点击“我也申请”展示企业微信二维码，并记录行为。
3. 前台只展示后台已上架内容。

### 7.9 资讯

前台列表按图文卡片展示。

字段展示：

1. 标题
2. 封面图
3. 更新时间或发布时间
4. 摘要

交互：

1. 点击卡片进入简单详情页。
2. 详情页显示普通正文文本。
3. 详情页底部保留“领取方案”按钮，点击展示企业微信二维码并记录行为。
4. 第一版不做富文本。

### 7.10 个人中心

默认入口：

1. 关于我们
2. 使用条款
3. 隐私协议
4. 软件版本
5. 联系我们
6. 退出登录

要求：

1. 不展示手机号。
2. 底部 CTA 展示企业微信二维码，并记录行为。
3. 企业、隐私、协议信息第一版使用演示占位文案，后续替换正式资料。

## 8. 后台管理需求

### 8.1 后台登录

1. 后台使用账号密码登录。
2. 管理员密码使用 BCrypt 加密存储。
3. 登录成功后保持 8 小时。
4. 退出登录后 token 失效。
5. 第一版不做登录验证码和登录失败锁定。

### 8.2 管理员管理

老板账号可进行：

1. 新增管理员。
2. 禁用管理员。
3. 重置普通管理员密码。

普通管理员不可进入管理员管理页。

第一版不做删除管理员账号。

### 8.3 客户与线索管理

后台默认首页为线索管理。

列表展示规则：

1. 默认按线索记录展示。
2. 同一手机号多条线索可折叠查看历史记录。
3. 仅登录未提交资料的客户也展示为“仅登录客户”。
4. 仅登录客户不创建正式债务线索，但后台列表和导出中可见。

默认筛选条件：

1. 手机号或称呼关键词
2. 地区
3. 线索类型
4. 债务类型
5. 债务金额区间
6. 创建时间范围
7. 是否查看过企业微信二维码

### 8.4 Excel 导出

导出规则：

1. 先筛选，再导出。
2. 支持自定义勾选导出字段。
3. 默认字段全选。
4. 文件名自动带日期，例如 `线索导出-20260510.xlsx`。

默认可选字段：

1. 手机号
2. 称呼
3. 地区
4. 债务金额
5. 债务类型
6. 债务描述
7. 线索创建时间
8. 首次登录时间
9. 最近行为
10. 是否查看企业微信二维码
11. 最近查看企业微信二维码时间

### 8.5 首页素材管理

后台只管理截图中的固定入口，不支持自由新增、删除、排序。

默认可管理：

1. 顶部视频
2. 视频封面图
3. AI 债务咨询图
4. 网贷利率计算器图
5. 规划优化债务图
6. 统一企业微信二维码

服务流程第一版固定展示，不做后台配置。

### 8.6 视频管理

1. 首页视频支持后台上传。
2. 第一版只支持 `mp4`。
3. 视频点击后弹窗播放。
4. 未上传视频时显示封面或占位，并提示“视频即将上线”。

### 8.7 资讯管理

支持：

1. 新增
2. 编辑
3. 删除
4. 上下架

字段：

1. 标题
2. 封面图
3. 摘要
4. 正文文本
5. 发布时间
6. 排序值
7. 是否上架

删除采用逻辑删除。

### 8.8 成功案例管理

支持：

1. 新增
2. 编辑
3. 删除
4. 上下架

字段：

1. 称呼
2. 脱敏手机号
3. 逾期平台
4. 逾期金额
5. 处理方案
6. 封面或头像
7. 详情文本
8. 排序值
9. 是否上架

删除采用逻辑删除。

### 8.9 操作日志

第一版只记录关键操作日志。

记录范围：

1. 管理员登录
2. 新增管理员
3. 上传素材
4. 新增、编辑、删除资讯
5. 新增、编辑、删除案例
6. 导出 Excel

第一版不做复杂审计页面，后续需要再扩展。

## 10. 文件上传规则

### 10.1 图片

- 支持类型：jpg、jpeg、png、webp
- 单张最大：20MB

### 10.2 视频

- 支持类型：mp4
- 单个最大：500MB

### 10.3 存储

1. 文件保存到本地可配置上传目录。
2. 数据库存储访问地址和原始文件名。
3. 新文件替换旧素材后立即生效。
4. 旧文件第一版先保留，不自动删除。

## 11. 安全与合规

### 11.1 安全

1. 后台接口需要管理员登录态保护。
2. 管理员密码 BCrypt 加密。
3. 上传接口仅管理员可调用。
4. 导出接口仅管理员可调用。
5. 客户登录态使用服务端生成 token，前端本地保存。
6. 开发环境允许前端 dev server 跨域访问后端。
7. 演示打包后尽量同域访问。

### 11.2 合规提示

第一版使用轻量合规提示：

1. 服务效果需结合个人情况评估。
2. 用户信息仅用于回访沟通和初步评估。
3. 不写“保证减免”“保证停催”“保证延期”等绝对承诺。
4. 企业名称、备案号、隐私协议、使用条款第一版使用演示占位文案。
